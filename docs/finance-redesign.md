# BankApp 重设计规格文档

> 设计时间：2026-05-21，修订 2026-05-23
> 参考产品：MoneyThings（多账户 + 多币种 + 3层分类 + iOS 原生风格）
> 目标：实用记账 + 角色陪伴评论，iOS 原生风格 + 柔化处理

---

## 一、整体结构

### 底部三栏 Tab

| Tab | 内容 |
|-----|------|
| **资产** | 总资产 + 周/月/年/全部四档趋势 + 账户卡片列表（按类型分组）；右上角齿轮进设置（二级页面） |
| **交易** | 收支汇总卡片 + 八卦情报卡片 + 按日分组流水列表 + 右下角 ＋ 记账按钮；点击交易进详情（二级页面） |
| **分析** | 上半：周/月/年切换 + 分类饼图 + 百分比列表；下半：TA 读（选角色评价，共用时间维度） |

### 二级页面

**交易详情页**
- iOS form 风格：金额 / 账户 / 分类 / 时间 / 备注
- 底部朋友圈式角色评论区：点"💬 让大家看看"手动刷新，各角色冒出短评
- 评论不自动生成，省 API 调用

**设置页**
- 从资产页右上角齿轮进入
- 账户管理（增删改）、分类管理（3层）、币种设置

## 二、数据模型

### 账户（FinanceAccount）

```typescript
interface FinanceAccount {
  id: string;
  name: string;               // "招商储蓄卡"、"Chase Checking"
  type: 'checking' | 'savings' | 'credit' | 'cash';
  currency: string;           // 'CNY' | 'USD' | 'JPY' | ...
  initialBalance: number;     // 手动填入的初始余额
  color: string;              // 卡片颜色
  icon?: string;              // emoji
  isArchived?: boolean;
}
```

**余额计算方式：** `当前余额 = initialBalance + 所有收入 - 所有支出 - 转出 + 转入`
（信用卡余额为负数表示欠款，initialBalance 通常填 0 或当前欠款额的负值）

### 分类（FinanceCategory）—— 3层层级

```typescript
interface FinanceCategory {
  id: string;
  name: string;
  icon?: string;              // emoji
  color?: string;
  parentId?: string;          // null = 一级，否则指向父级 id
  // 层级: 一级(餐饮) → 二级(外卖) → 三级(美团)
}
```

**默认预设分类：**
- 餐饮 → 外卖、堂食、咖啡、超市零食
- 交通 → 地铁/公交、打车、加油、机票
- 购物 → 服饰、电子、日用品
- 娱乐 → 订阅/会员、游戏、电影
- 医疗
- 学习
- 转账（系统保留，不显示在支出统计里）
- 收入 → 工资、奖学金、转账收入、退款

### 交易（FinanceTransaction）

```typescript
type TxType = 'expense' | 'income' | 'refund' | 'transfer';

interface FinanceTransaction {
  id: string;
  type: TxType;
  amount: number;             // 始终为正数
  currency: string;           // 记录时的币种
  accountId: string;          // 来源账户
  categoryId: string;
  note: string;
  timestamp: number;
  dateStr: string;            // 'YYYY-MM-DD'

  // 仅 type='transfer' 时有效
  toAccountId?: string;
  toAmount?: number;          // 跨币种转账时目标金额
  toCurrency?: string;

  // 仅 type='refund' 时有效
  refundForTxId?: string;     // 原始交易 id（可选）

  // 角色评论（朋友圈式，手动触发后缓存）
  charComments?: { charId: string; comment: string; timestamp: number }[];
}
```

**信用卡还款**：type='transfer'，fromAccount=储蓄卡，toAccount=信用卡
**退款**：type='refund'，金额为正，进入来源账户，统计时不计入支出

## 四、删除的上游功能

以下 gamified 功能全部删除：
- AP 系统（行动点数、每日预算 mechanic）
- 招聘/解雇/休息店员
- 解锁食谱
- 多房间 Dollhouse
- 店铺等级、人气值
- 咖啡馆作为顶部 banner（改为交易页情报卡片点入的二级页面）

## 五、UI 风格

**iOS 原生骨架 + 柔化处理：**
- 背景：浅暖灰白 `#F8F7F4`（不是纯白也不是冷灰）
- 卡片：白色 `bg-white` + 柔阴影 `shadow-sm` + 大圆角 `rounded-2xl`
- 数字：系统字体加粗，清晰可读
- 颜色：温润色调（收入用青绿而非纯绿，支出用柔粉红而非纯红）
- 动效：弹性 `cubic-bezier(0.34, 1.56, 0.64, 1)`
- 整体感觉：MoneyThings 的信息密度 + iOS 备忘录的温度 + 聊天页的圆角柔和感

## 六、实现顺序

1. **数据模型** — `utils/financeDb.ts` + `types.ts` 新增类型（FinanceAccount / FinanceCategory / FinanceTransaction）
2. **资产页** — 总资产 + 趋势图 + 账户列表
3. **交易页** — 收支汇总 + 流水列表 + 记账表单（＋按钮）
4. **交易详情页** — iOS form + 朋友圈评论（手动触发）
5. **分析页** — 饼图 + 百分比列表 + TA 读区域
6. **八卦情报** — 轻量卡片 + 详情页（改为每日免费，删 AP）
7. **设置页** — 账户管理 + 分类管理
